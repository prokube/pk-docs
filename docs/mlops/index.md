# MLOps

MLOps is the prokube product track for developing, training, tracking, and serving machine learning models.

It covers the classic data science and machine learning lifecycle on top of the shared platform foundation.

We are currently moving the prokube documentation to this site. In the meantime, please use [docs.prokube.ai](https://docs.prokube.ai/latest/) for MLOps-related documentation. These pages will be migrated here soon.

## Core Capabilities

- Lab environments for interactive development and pipeline authoring
- Kubeflow Pipelines for workflow automation
- MLflow for experiment tracking and model registry workflows
- KServe and related runtimes for model serving
- GPU support for training and inference workloads
- Storage integration for datasets and model artifacts

## Shared Services

MLOps uses the same foundation as AgentOps: workspaces, identity, RBAC, storage, observability, and Kubernetes operations.

Some capabilities, especially Agent Gateway, can serve both tracks. For example, Agent Gateway can protect classic model-serving endpoints as well as agent-oriented APIs.
